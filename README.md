# Cisco Packet Tracer – Spanning Tree Protocol (STP) Labs

Three Cisco Packet Tracer labs exploring Spanning Tree Protocol (IEEE 802.1D), from a basic three-switch loop to a complex nine-switch topology with custom bridge priorities.

## Labs

| Lab | Description | Link |
|---|---|---|
| **Lab 1** | 3 switches in a triangular topology with redundant links — basic Root Bridge election and port blocking. | [lab1/](./lab1) |
| **Lab 2** | 2 switches connected by two redundant links — forwarding/blocking states and automatic failover. | [lab2/](./lab2) |
| **Lab 3** | 9 switches with multiple redundant links and custom bridge priorities — advanced convergence and Root Bridge control. | [lab3/](./lab3) |

Each lab folder includes:
- `stp1.pkt` / `stp2.pkt` / `stp3.pkt` — the Packet Tracer project file
- `screenshots/` — network topology diagram
- `README.md` — full write-up: objectives, topology, verification commands, expected results

## Skills Demonstrated
- Spanning Tree Protocol (IEEE 802.1D)
- Root Bridge election and bridge priority tuning
- Root Port / Designated Port / Blocking Port identification
- Layer 2 loop prevention
- Network redundancy and automatic failover
